新鲜劲过了后，就觉得少了点意思，现在用notion做为blog平台了，省心省力，就是被别人发现的机会更少了，即使我开启了全网公开，域名我也没修改成功，不太方便推广，但本着方便自己的原则，我是为自己记录，所以不想将就了，notion除了对中文字体支持不是很好，其他功能都没的说，下载文件也方便，插入图片也不用图床了，总之是各种省心。

欢迎光临我的blog：https://www.notion.so/xuanxi/b40d8460c4334e1a9996aae18d24235e?v=4f4a45378e2d40a380abd1a475778568

有些长，生成了一个短网址：https://j.mp/38aAi4Y

据说可以通过cloudflare绑定自己的域名的，尝试了没成功，URL定向，后者挺不错的看上去，可是也失败了，哎，不管了。

「书籍推荐」是作为激励我阅读的一个长期工程，目前我感兴趣的是社科类，因为用kinde读而且喜欢azw3格式，所以对于我分享的书籍，都是用epub、azw3、mobi格式都有的。

经过我的尝试，有以下几种分享的途径：

  1. 百度网盘共享文件夹，加入后可以共同管理。缺点是速度一般，但也够用
  2. OneDrive共享文件夹，OneDrive对我来说是个人同步用的好工具，但分享在国内就捉襟见肘了
  3. QQ分享群：152772001，优点是速度飞起，而且方便交流，但估计也没人加群，这个上传也不如上面两个方便
  4. github的库，是我本来设想的，这个blog本身就是依托github搭建的，也算是同气连枝，可惜的是经过尝试发现一般情况下下载不了，要修改hosts文件才行，一般的话除非是专业人士和我这种没事找事的，恐怕用不到。

以上四种方法，都放在侧边栏的友链里了，需要自取，欢迎加入讨论。

记录一下常用的命令：

  • cd ..
  • cd blog
  • hexo new “new post.md”
  • hexo new draft “new draft”
  • hexo publish “new draft”
  • hexo cl
  • hexo g
  • hexo s
  • hexo d
  • git add .
  • git commit -m “message”
  • git push -u origin master
